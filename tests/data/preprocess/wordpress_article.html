Why Async Python Matters

By Andrew B. May 1, 2026

Python's asyncio module has been part of the standard library for over a decade, and it has shaped how high-concurrency Python services are built.

For workloads that are network-bound — talking to databases, caches, or external HTTP services — an event-loop-based concurrency model can be dramatically more efficient than threads.

This post walks through three common patterns: structured concurrency with asyncio.TaskGroup, fan-out with gather, and bounded concurrency with Semaphore.

Structured concurrency

The 3.11 release introduced TaskGroup, which gives you a context manager that owns its child tasks and propagates exceptions cleanly when any of them fail.

async with asyncio.TaskGroup() as tg:
    tg.create_task(fetch(url1))
    tg.create_task(fetch(url2))

When threads still win

If your workload is CPU-bound, async won't help. The GIL still serializes Python bytecode, and you're better off with multiprocessing or a process pool executor.
