## Gap (why Yosoi lost this on the hotpath)
Yosoi's deterministic replay runtime already exists and is LLM-free (yosoi/core/replay/runtime.py:58 execute_plan, with a full act dispatch table incl. TELEPORT at :217-226), but it is completely unwired from the production fetch/scrape path, and the public API is shaped wrong for Nimbal's hotpath. Three concrete shape mismatches: (1) The fetch substrate takes raw JS strings, not a ReplayPlan — _VoidCrawlFetcher.fetch(url, action_scripts: dict[str,str]) (voiddriver.py:177-184, :346 _eval_action_scripts) does tab.goto(url) then evals strings; execute_plan is never called on the hotpath. mcp_orchestrator._lesson_to_selector_map deliberately does NOT execute lesson.replay_plan (mcp_orchestrator.py:271-275) and _navigate_plan only emits a bare navigate (mcp_orchestrator.py:56-66 — 'intentionally unwired, not dead code'). (2) The discover-once unit is keyed by LessonKey(domain, contract_signature, page_profile) (replay.py:139-151) — a per-DESTINATION-domain cache. Nimbal's hotpath is the inverse: one program per ENGINE/TOOL host (google.com, similarweb.com, semrush.com) replayed across N target domains with a parameter substituted — the {d} in spec['url'].format(d=inp.domain) (_real.py:109) and the SERP query q (_real.py:78-79). ReplayPlan/ReplayAct have no parameter/template slot — act.url and act.script are frozen strings (replay.py:64-76). serp_contracts.py:1-8 explicitly documents fighting the per-domain cache. (3) Teleport and captcha are nowhere in the fetch path: grep of yosoi/core/fetcher/*.py for detect_captcha|teleport|set_geolocation returns ZERO hits. Teleport exists only as an unwired ReplayAct executor (_exec_teleport runtime.py:201, _teleport :360 calling tab.set_geolocation/set_timezone/set_locale); captcha detection lives only in Nimbal's raw path (page.detect_captcha() at _real.py:116, scrape_authority.py:102). So Nimbal cannot express 'teleport to Arlington VA, then run the SERP program' or 'detect+pause on Turnstile' through ys.scrape at all.

## Proposal
Wire the EXISTING execute_plan into the fetch path and add a parametrized, per-engine replay API. Four moves, smallest-diff first:

1. PARAMETRIZE the replay plan (model-only change). Add an optional `params: dict[str,str]` to execute_plan and substitute into act.url/script/text/targets at dispatch time (str.format_map with a missing-key guard → fail-fast). This makes one discovered DiscoveryLesson for similarweb.com replay across all 30+ target domains by passing params={'d': domain}; one google.com SERP lesson replays across all queries by params={'q': query}. No new act kinds, preserves CAS-87 (substitution is pure string templating, no model call).

2. SPLIT the lesson key so 'discover-once-per-engine' is first-class. Add `param_keys: tuple[str,...]` to LessonKey and make the cache key (engine_host, contract_signature) — NOT target domain. The target domain becomes a replay-time param, not part of identity. This is the inversion Nimbal needs and what serp_contracts.py was fighting.

3. WIRE execute_plan into _VoidCrawlFetcher. New fetch path: if a DiscoveryLesson is present for (engine_host, contract_sig), acquire a pooled tab and call execute_plan(tab, lesson.replay_plan, params=...) instead of goto+eval_js strings. The EVAL acts carry the selector-driven extraction; output_field captures map straight to the contract fields the yosoi_bridge adapter already consumes.

4. FOLD W1 (captcha) + W3 (teleport) into the plan, not the call site. Teleport is already a ReplayAct (TELEPORT) — discovery emits a leading TELEPORT node from params={'lat','lng','tz','locale'}. Captcha becomes a ReplayCondition gate (new AssertKind.CAPTCHA_CLEAR) on assess: if Turnstile present, the runtime raises ReplayExecutionError (fail-fast) and the orchestrator rotates proxy/profile via tenacity Retrying — NOT a solve-in-hotpath, NOT a model call.

Public API shape (new module yosoi.serp or extend api.py):
- `program = await ys.discover_program(engine_url_template, contract, params_example, model=...)` → returns/persists a DiscoveryLesson keyed by engine host. Runs ONCE per engine/tool.
- `rows = await ys.replay(program, params={'d': domain}, teleport=Teleport(...), captcha='rotate')` → deterministic, LLM-free, returns list[dict] for the adapter.
- A `ReplaySession` context manager that holds ONE pooled tab and replays many params against the same engine host (mitigates the multi-tab antibot risk — Nimbal's real_serp already reuses one page across queries at _real.py:74-86).

## Risks
- Teleport timeout: set_geolocation/set_timezone via CDP can hang on a cold pooled tab (voiddriver.py:162 already notes cold-tab transient timeouts). Mitigation: TELEPORT node carries timeout_ms + tenacity Retrying(stop_after_attempt(2), wait_exponential) around tab acquire — no raw sleep loop
- Antibot in hotpath / multiple tabs: parallelizing 30+ domains x 4 tools = many simultaneous landings on similarweb/semrush → rate-limit/Turnstile storm. Mitigation: ReplaySession holds ONE tab per engine host and serializes params (mirrors _real.py:74-86 single-page reuse); BrowserPool jitter already staggers (voiddriver.py:201-205). Cap concurrency per engine host, not global
- Non-determinism leak: _bind via str.format_map is safe, but if discovery ever emits an EVAL whose script embeds a param INSIDE a JS string literal, format could corrupt JS braces. Mitigation: JSON-encode param values + use a non-{} delimiter for script params, or validate at discover_program time that scripts have no bare {}
- Captcha-as-condition could fail-open: if AssertKind.CAPTCHA_CLEAR is only checked on assess (pre-act) it can miss a challenge injected after navigation. Mitigation: also gate on expect and re-check post-EVAL when output_field is empty (the _real.py:121 'cap and not metrics' heuristic, made structural)
- Stale-lesson blast radius: one engine reskin now invalidates a program replayed across ALL domains (vs per-domain today). Mitigation: VerifyReport score threshold (replay.py:182 LessonValidation, base.py:217 replay_verify_threshold) auto-marks STALE → re-discover that one engine; this is strictly better than 30 silent empties
- Logged-in-profile coupling: real_* depends on a live Chromium profile (_real.py:13-14 SEMrush login). ReplaySession must thread extra_args/profile through to BrowserConfig or the authority programs lose their authenticated views
